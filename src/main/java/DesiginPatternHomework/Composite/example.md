# 组合模式 (Composite Pattern)

## 题目
设计一个文件系统，支持文件和文件夹的树形结构，能够统一处理单个文件和文件夹的操作（如打印结构、计算大小）。

## 模式说明
组合模式将对象组合成树形结构来表示"部分-整体"的层次结构，使得用户对单个对象和组合对象的使用具有一致性。

## 代码结构

### 核心类
- **FileSystemItem** (接口): 组件接口，定义文件和文件夹的公共操作
    - `print(String prefix)`: 打印文件系统项
    - `getSize()`: 获取大小

- **File** (叶子节点): 表示文件，实现 FileSystemItem 接口
    - 包含文件名和大小
    - 实现打印和获取大小的方法

- **Folder** (容器节点): 表示文件夹，实现 FileSystemItem 接口
    - 包含文件夹名和子项列表
    - 递归打印子项结构
    - 递归计算所有子项的总大小

### 使用示例
```java
File file1 = new File("file1.txt", 128L);
Folder folder1 = new Folder("folder1", List.of(file1, file2));
folder1.print(""); // 打印文件夹结构及大小
```

## 优势
- 统一处理单个对象和组合对象
- 简化客户端代码，无需区分叶子节点和容器节点
- 易于扩展，可方便地增加新的组件类型
- 符合开闭原则
